Standard Cache Management

Summary

This module manages the Standard Cache Policy of XCL Web Application Platform.

The standard policy cache files for every users group except the webmaster group.

What is this?

XCL introduced a new Cache Policy concept called Standard Cache.

Its purpose is to reduce the time it takes to perform certain scenarios

caching files of modules and blocks.

The cache feature saves the content to files, if setting cache active.

And, if cache files are available, XCL prevents the extra logic SQL of blocks and modules.

This is a useful and powerful feature to speed up a web page load for all site visitors.

Understanding Group Policy Caching

For security reasons, the Standard Cache Policy should prevent the caching of files for administrators' access.

Otherwise, the files cached could be accessed by anonymous users.

Also, to prevent the cache to grow larger with too many files in your server,

the cache of modules can be enable or disable from General Settings.

Best practices for cache control settings

Set the time that the module or block should keep the files in cache.

Therefore, the Standard Cache Policy can cache many files, so it features a "cache clear block".

The cache clear block
Summary

This block is not visible and can delete all of cached files, including extra cache files, with interval timer.

Options

Interval timer (min)
Specify the interval.
